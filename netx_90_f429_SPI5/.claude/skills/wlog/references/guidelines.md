# 작업 로그 작성 가이드

## 작업 로그의 목적
- 작업 내역의 체계적 기록 및 추적
- 프로젝트 진행 상황 모니터링
- 문제 해결 과정 문서화
- 팀 간 커뮤니케이션 개선
- 향후 참고 자료로 활용

## 작성 시 고려사항

### 1. 명확성
- 제3자가 읽어도 이해할 수 있도록 명확하게 작성
- 전문 용어는 필요시 설명 추가
- 구체적인 수치와 데이터 포함

### 2. 완결성
- 작업의 시작부터 끝까지 전체 흐름 기록
- 의사결정 과정과 근거 명시
- 관련 리소스 및 참고 자료 링크

### 3. 시간순 기록
- 작업 순서대로 기록하여 흐름 파악 용이
- 중요한 시점(마일스톤) 명시
- 시간 관련 정보 포함 (시작 시간, 완료 시간 등)

### 4. 문제와 해결책
- 발생한 문제를 상세히 기록
- 시도한 해결 방법들 나열
- 최종 해결책과 그 근거 설명
- 재발 방지 대책 포함

## 섹션별 작성 팁

### 작업 개요
- 날짜, 담당자, 프로젝트명 등 기본 정보
- 작업의 목적과 배경 간략히 설명

### 수행 작업
- 구체적인 작업 항목을 순서대로 나열
- 각 작업에 소요된 시간 기록 (선택사항)
- 작업의 우선순위나 중요도 표시 (선택사항)

### 진행 상황
- 체크리스트 형태로 완료/미완료 구분
- 진행률(%)을 표시하면 더욱 명확
- 예상 완료 시점 추가 가능

### 이슈 및 해결
- 문제 발견 시점 기록
- 증상과 원인을 분리하여 설명
- 해결 과정을 단계별로 기술
- 참고한 문서나 도움을 받은 사람 언급

### 다음 작업
- 우선순위를 고려한 작업 순서
- 예상 소요 시간 추가 가능
- 필요한 리소스나 협조 사항 명시

## 파일명 규칙
- 형식: `YYYYMMDD_HHMMSS_work_log.md`
- 예시: `20251102_143025_work_log.md`
- 날짜와 시간이 포함되어 정렬과 검색이 용이

## 저장 위치
- 프로젝트별 디렉토리에 저장
- 팀 공유 공간에 업로드
- 버전 관리 시스템에 커밋 고려

## 주의사항
- 민감한 정보(비밀번호, API 키 등)는 기록하지 않음
- 개인 공격이나 비난은 배제
- 객관적이고 중립적인 톤 유지
- 정기적으로 작성하여 누락 방지
